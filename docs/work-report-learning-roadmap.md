# 업무보고 이미지 매칭 개선 로드맵

최종 업데이트: 2026-04-18

## 목표

업무보고 미리보기에서 이미지 매칭이 애매한 경우만 사람 검토를 요청하고, 그 결과를 구조화된 피드백으로 쌓아 장기적으로 매칭 정확도를 높인다.

## 개발 순서

1. 완료: 수동 이미지 재배정과 PDF 반영
- 미리보기에서 사진별로 연결 작업과 단계값을 직접 수정할 수 있게 했다.
- 수정 결과는 PDF 생성 시 그대로 반영된다.

2. 완료: 수동 피드백 저장
- 사람이 수정하거나 확정한 결과를 `/api/ai/work_report/feedback`로 저장한다.
- 현재는 감사로그 기반으로 누적한다.

3. 완료: 애매한 이미지 검토 큐와 추천 후보 3순위
- 점수 차이가 작거나 단서가 약한 사진만 따로 모은다.
- 각 사진에 대해 추천 후보 3개와 근거를 보여준다.
- 현재 선택 유지도 사람 판단으로 확정할 수 있다.

4. 다음: 전용 피드백 저장소 분리
- 감사로그와 별도로 `work_report_image_feedback` 성격의 전용 테이블을 둔다.
- 원래 예측, 후보 점수, 최종 선택, 테넌트, 모델 버전, 검토 시각을 구조화 저장한다.

5. 다음: 테넌트별 즉시 보정 레이어
- 저장된 피드백을 바탕으로 특정 테넌트에서 자주 나오는 패턴을 재랭킹에 즉시 반영한다.
- 전체 모델 재학습 대신 안전한 가중치 보정부터 적용한다.

6. 다음: 오프라인 평가와 배포 기준
- top-1 정확도, top-3 포함률, 미매칭 오판률, 사람 개입률을 측정한다.
- 개선안은 평가셋 기준으로 좋아질 때만 배포한다.

7. 이후: 학습 데이터셋과 모델 개선
- 충분한 피드백이 쌓이면 few-shot 예시, 재랭커, 또는 별도 파인튜닝용 데이터셋으로 확장한다.

## 현재 구현 메모

- 검토 큐는 `work_report_service`의 점수 기반 후보 계산을 사용한다.
- 사람이 추천 후보를 누르거나 현재 선택을 확정하면 해당 이미지는 검토 큐에서 빠진다.
- 저장되는 피드백은 `재배정`, `미매칭 처리`, `단계 수정`, `현재 선택 확정`으로 구분한다.

## 운영 메모

- 즉시 온라인 재학습은 하지 않는다.
- 사람이 한 번 잘못 고른 값이 전체 매칭을 오염시키지 않도록, 우선은 저장과 재랭킹 분리 전략을 유지한다.
